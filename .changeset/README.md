# Changesets

Add one Markdown file here for every release-relevant change.

Each file selects `patch`, `minor` or `major` explicitly for `homeassistant_custom_refrigerator_card` and contains the release-note text.

CI-only and documentation-only changes do not require a Changeset.

Jenkins creates the version pull request, then creates the tag and publishes the immutable GitHub Release after all main quality gates pass.
